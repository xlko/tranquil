GIS Reporting mit R

April 22, 2018 in R GIS

Dieser Beitrag zeigt wie man mittels R ein GIS-Reporting erstellt und kann als Ergänzung zum letzten Beitrag betrachtet werden. Geographische-Informationssysteme (GIS) werden verwendet, um räumliche Informationen zu verarbeiten und abzubilden. Ein Großteil der offenen Datensätze enthalten Ortsangaben, die sich mit GIS verknüpfen lassen, so dass die Ortsangaben kartographisch genutzt werden können. R stellt zahlreiche Pakete zu Verfügung mit denen sich ein GIS umsetzen lässt.

Daten

Im letzten Post wurden die Daten der Schülerzahlen in NRW verwendet, um ein Powerpoint-Reporting zur Schüler-Lehrer-Relation zu erstellen, die in der Variablen “betreuung” festgehalten wurde. Nach dem Datenmanagement lag folgender Datensatz vor:

## Loading required package: htmlwidgets

Eine weitere Möglichkeit die Daten darzustellen, ist die Visualisierung in einer Karte. Öffentliche Daten in Deutschland sind häufig, aufgrund von Anonymisierung, über Gebietseinheiten (Bundesländer, Landkreise, usw.) aggregiert. Diese Aggregation kann in Chorophletenkarten visualisiert werden. Im Datensatz liegen in den Variablen “regierungsbezirk” und “kreis” räumliche Informationen vor. Diese alleine reichen jedoch nicht für eine kartographische Darstellung aus. Es fehlen Informationen zu Flächeneinheiten und deren Koordinatenbezug, sog. Shapefiles. Eine Verknüpfung der Daten mit diesen shapefiles erlaubt das Erstellen von Chorophletenkarten.

Kartenbezug

Eine nicht-proprietäre Variante dieser Shapefiles sind Dateien im .geojson Format und können für NRW im OpenDataLab bezogen werden. Mit dem Paket geojsonio lässt sich dieses Datenformat in R einlesen. Darüber hinaus kann mit dem leaflet Paket über javascript eine interaktive Karte eingebunden werden auf der sich das shapefile plotten lässt.

library(geojsonio, warn.conflicts=FALSE)
library(leaflet)
nrw <- geojson_read("/home/alex/blogdown/rscriptpost/landkreise_simplify200.geojson",
                    what = "sp")
## Einbinden des shapefile in die Karte
karte.nrw <-  leaflet(nrw, height=500) 
## Definition der Anzeige des Kartenausschnitts mittels Koordinaten
karte.nrw <-  setView(karte.nrw, 7.61,51.5,   zoom=0) 
karte.nrw <-  fitBounds(karte.nrw, 5.5, 50.26, 9.51, 52.55)
karte.nrw <-  addTiles(karte.nrw)
## Einsetzen der Fälchen in den Karten layer
karte.nrw <-  addPolygons(karte.nrw)
## Darstellung der Karte
karte.nrw

Die geojson Datei wird nach dem Einlesen als ein S4-Objekt ausgegeben und liegt nicht in einer klassischen Datenmatrix vor. Daten und Geoinformationen befinden sich in getrennten Einheiten -sog. Slots- und müssen dementsprechend bearbeitet werden. Weitere Erläuterungen zu S4-Objekten finden sich hier. Ein Blick in den Daten-Slot des shapefiles zeigt, ob es Möglichkeiten des Matchings mit dem Schuldatensatz gibt:

Matching der Daten

Über die Variablen schulen.nrw$kreis_text und nrw$GEN lässt sich ein Matching durchführen. Eine Anpassung der Textvariablen ist jedoch notwendig:

## Erstellen einer neuen Textvariable ohne Attribut "Kreis" und "Krfr"
schulen.nrw$kreis_match     <- sub('(^Krfr. Stadt\\s|^Kreis\\s)', '', schulen.nrw$kreis_text)
## Aggregieren von Betreuungsverhältnis über Landkreise als neue Variable
schulen.nrw$betreuung_kreis <- ave(schulen.nrw$betreuung,
                                   schulen.nrw$kreis_match,
                                   FUN = function(x){mean(x, na.rm=TRUE)})
## Datensatz bestehend aus den zwei neuen Variablen
schulen.match <- schulen.nrw[names(schulen.nrw) %in% c('kreis_match', 'betreuung_kreis')]
## Entfernen der Kreis-Duplikate
schulen.match <- schulen.match[!duplicated(schulen.match$kreis_match), ]
schulen.match <- schulen.match[order(schulen.match$kreis_match), ]
nrw@data      <- nrw@data[order(nrw@data$GEN), ]
## Abbruch des Skripts wenn beide Variablen nicht identisch sind
stopifnot(nrw@data$GEN == schulen.match$kreis_match)
## Falls identisch dann als neue Variablen in den Datenslot des kartendatensatzes hinzufügen
nrw@data$betreuung  <- schulen.match$betreuung_kreis

Die Verbindung der unterschiedlichen Slots im Shapefile wird durch die Zeilnamen gewährleistet. Die Zeilnamen verbinden die Datenattribute mit den räumlichen Infos. Es ist daher besonders wichtig,dass die Zeilnamen über alle Slots hinweg in gleicher Reihenfolge vorliegen. Ansonsten werden den Daten die falschen räumlichen Informationen zugeordnet.

## Die Daten werden jetzt nach der ursprünglichen Reihenfolge geordnet
nrw@data <- nrw@data[order(as.numeric(rownames(nrw@data))), ]

Darstellung der Choroplethenkarte

Nachdem die Daten nun in der richtigen Reihenfolge vorliegen, kann jetzt die Chorophletenkarte mit den unterschiedlichen Anteilen in der Betreuungsrelation von Schülern pro Lehrer erstellt werden.

karte.nrw <-  leaflet(nrw) 
## Definition der Anzeige des Kartenausschnitts mittels Koordinaten
karte.nrw <-  setView(karte.nrw, 7.61,51.5,   zoom=1) 
karte.nrw <-  fitBounds(karte.nrw, 5.5, 50.26, 9.51, 52.55)
karte.nrw <-  addTiles(karte.nrw)
## Javascript labels mittels C# Wrapper erstellen
labels <- sprintf("<strong>%s</strong><br/>%g SuS pro Lehrkraft",
                  nrw$GEN,
                  round(nrw$betreuung,2))
## Ablegen der Labels in eine Liste
labels    <- lapply(labels, htmltools::HTML)
## Funktion für Farben Intervalle in Chorophletenkarte
farben    <-    colorBin("Reds", domain = nrw$betreuung, bins=4)
## Plotten der Flächenanteile und hinzufügen einer Legende
karte.nrw <- addPolygons(karte.nrw,
                          fillColor = ~farben(nrw$betreuung),
                          weight = 2,
                          opacity = 1,
                          color = "white",
                          dashArray = "3",
                          fillOpacity = 0.7,
                          highlight = highlightOptions(
                          weight = 5,
                          color = "#666",
                          dashArray = "",
                          fillOpacity = 0.7,
                          bringToFront = TRUE),
                          label = labels,
                          labelOptions = labelOptions(
                          style = list("font-weight" = "normal", padding = "3px 8px"),
                          textsize = "15px",
                          direction = "auto")) 
karte.nrw <-  addLegend(karte.nrw,
                         pal = farben,
                         values = ~nrw$betreuung,
                         opacity = 0.7,
                         title = NULL,
                         position = "bottomright")

karte.nrw

Die Visualisierung des Betreuungsverhältnisses mittels Chorophletenkarte lässt auf einen Blick erkennen, in welchem Kreis in NRW Lehrer besonders viel bzw. wenig Schüler betreuen. Gerade um einen ersten Eindruck über die unterschiedlichen Anteile und den räumlichen Bezug zu gewinnen ist diese Darstellungsform besonders geeignet.
